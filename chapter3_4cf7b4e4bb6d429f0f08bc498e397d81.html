第3章：こだわりデザイン編 〜『プロ』の風格を出す〜

「黒すぎる画面」からの脱却

アプリは動きました。でも、画面を見て、私は少し違和感を覚えました。

「なんか、真っ黒すぎない？」

ダークモードに設定した画面は、確かに目に優しいはずでした。でも、この真っ黒な背景は、何か違う。まるで、宇宙の闇に飲み込まれそうな、そんな気持ちになりました。

「もっと、柔らかい色にできないかな」

私は、AIにこう伝えました。

画面の背景色を、真っ黒に近いグレーではなく、 もう少し柔らかいダークグレー（#1e1e1e）に変更してください

AIは、すぐに画面の部品（page.tsx）を修正してくれました。

数秒後、画面をリロードすると、背景色が変わりました。

「あ、これだ」

真っ黒ではなく、少し明るいグレー。まるで、高級なノートパソコンの画面のような、上品な色合いでした。

目に優しい文字色

背景色を変えたら、次は文字色です。

「白すぎる文字も、目が疲れるかも」

私は、AIにこう伝えました。

文字の色も、完全な白ではなく、目に優しいオフホワイト（#e0e0e0）にしてください

AIは、またすぐに修正してくれました。

#e0e0e0 という色。これは、完全な白（#ffffff）よりも、少し柔らかい色です。長時間見ていても、目が疲れにくい、そんな優しい色でした。

「これで、長時間使っても大丈夫だ」

背景色と文字色を変えただけで、画面の印象ががらりと変わりました。まるで、安いアプリから、高級なアプリに生まれ変わったような、そんな感覚でした。

デザインに「意味」を持たせる

添削結果を「特別なもの」に

アプリは動きました。でも、添削結果が、ただ文字として表示されるだけでは、物足りませんでした。

「もっと、『これは特別な結果だ』という感じにできないかな」

私は、AIにこう伝えました。

添削結果を、白抜きのボックスで囲んで、 「特別な結果」であることを演出してください

AIは、画面の部品を修正して、添削結果を表示する部分に、背景色と枠線を追加してくれました。

css background: #1e1e1e; border: 1px solid rgba(255, 255, 255, 0.1);

この設定で、添削結果が、まるで高級なレストランのメニューのように、白いボックスの中に美しく収まりました。

本物の編集ツールのような高級感

画面を見て、私は感動しました。

「これ、本物の編集ツールみたい」

プロの編集者が使うような、高級なソフトウェアのような、そんな雰囲気が漂っていました。

でも、まだ物足りませんでした。

「もっと、行間を広げて、読みやすくできないかな」

「箇条書きの部分をもっと見やすくできないかな」

AIに伝えるたびに、画面は少しずつ、洗練されていきました。

  • 行間を広げる → leading-7 という設定を追加
  • 箇条書きを見やすく → インデントと間隔を調整
  • 枠線を追加 → 控えめな境界線で、背景との区別を明確に

こうして、何度もAIと対話を重ねながら、画面は「プロのツール」らしい風格を備えていきました。

世界への第一歩：GitHubとVercel

コードの「金庫」：GitHub

アプリが完成したら、次は「保存」です。

でも、ただ自分のパソコンに保存するだけでは、不安でした。

「もし、パソコンが壊れたら？」

「もし、コードを消してしまったら？」

そんな不安を解消してくれるのが、GitHubという「コードの金庫」です。

GitHubに保存する手順

  1. GitHubのアカウントを作る

    • ブラウザで https://github.com を開きます
    • 「Sign up」をクリックして、アカウントを作成します（無料）

  2. 新しい「金庫」を作る

    • GitHubにログインしたら、右上の「+」ボタンをクリック
    • 「New repository」を選択
    • リポジトリ名を入力（例：tensaku-kun）
    • 「Create repository」をクリック

  3. コードをアップロードする

    • 作成したリポジトリのページに、コマンドが表示されます
    • ターミナルで、そのコマンドを実行します

bash git init git add . git commit -m "初回コミット" git branch -M main git remote add origin https://github.com/あなたのユーザー名/tensaku-kun.git git push -u origin main

このコマンドを実行すると、あなたのコードが、GitHubという「金庫」に保存されます。

「これで、どこからでもアクセスできる」

私は、GitHubにコードをアップロードした瞬間、安心感に包まれました。もう、コードを失う心配はありません。

Webの「放送局」：Vercel

コードを保存したら、次は「公開」です。

自分のパソコンで動いているアプリを、世界中の人が使えるようにする。そのために必要なのが、Vercelという「Webの放送局」です。

Vercelで公開する手順

  1. Vercelのアカウントを作る

    • ブラウザで https://vercel.com を開きます
    • 「Sign Up」をクリックして、GitHubアカウントでログインします

  2. プロジェクトを追加する

    • ダッシュボードで「Add New...」→「Project」をクリック
    • GitHubのリポジトリ一覧から、先ほど作成したリポジトリを選択

  3. デプロイする

    • 「Deploy」ボタンをクリック
    • 数分待つと、あなた専用のURLが発行されます

「これが、私のアプリのURLだ」

画面に表示されたURLを見て、私は感動しました。

https://tensaku-kun-あなたのユーザー名.vercel.app

このURLを、世界中の人がアクセスできる。まるで、自分だけのテレビ局を手に入れたような、そんな気持ちでした。

ボタンをポチポチ押すだけの魔法

実は、Vercelでの公開は、思っていたより簡単でした。

「コードを書く必要はないの？」

最初は、そう思っていました。でも、実際には、ボタンをポチポチ押すだけでした。

  1. GitHubのリポジトリを選択
  2. 「Deploy」ボタンをクリック
  3. 数分待つ

これだけです。

「え、これで終わり？」

数分後、画面に「Deployment successful」という文字が表示されました。そして、URLが発行されました。

「本当に、これで世界中の人が使えるの？」

私は、そのURLをブラウザで開いてみました。すると、自分のアプリが、確かに動いていました。

「すごい...本当に動いてる」

パソコンを閉じても、スマホからでも、そのURLにアクセスすれば、アプリが使えます。まるで、魔法のような体験でした。

「自分専用」の証明：URLのカスタマイズ

最初のURL

Vercelで公開すると、最初は、こんなURLが発行されます。

https://tensaku-kun-あなたのユーザー名.vercel.app

「これでも動くけど、もっと短くできないかな」

「もっと、サービス名らしいURLにできないかな」

私は、AIに聞いてみました。

「URLを変更する方法はありますか？」

AIは、優しく教えてくれました。

「Vercelの設定で、カスタムドメインを設定できます。でも、もっと簡単な方法があります。プロジェクト名を変更すれば、URLも変わります」

サービス名に変更する

Vercelのダッシュボードで、プロジェクトの設定を開きました。

「Settings」→「General」→「Project Name」を変更します。

tensaku-kun という名前に変更すると、URLも自動的に変わります。

https://tensaku-kun.vercel.app

「これだ！」

短くて、覚えやすい。そして、サービス名がそのままURLになっている。

「これが、私のサービスだ」

私は、そのURLを、メモ帳に保存しました。そして、友達に送ってみました。

「これ、私が作ったアプリだよ」

友達が、そのURLを開いて、アプリを使っている様子を見て、私は誇らしい気持ちになりました。

「これが、私のサービスだ」

胸を張って、そう言えるようになった瞬間でした。

【コラム】デザインはAIに丸投げでいい

「かっこよくして」で通じる

デザインを変更する時、最初は、私は具体的な指示を出していました。

「背景色を #1e1e1e に変更してください」

「文字色を #e0e0e0 に変更してください」

でも、ある時、試しにこう言ってみました。

かっこよくしてください

すると、AIは、背景色や文字色、レイアウトを、自動的に調整してくれました。

「え、これでいいの？」

画面を見て、私は驚きました。確かに、かっこよくなっていました。

「プロっぽくして」でも通じる

次に、こう言ってみました。

プロっぽくしてください

すると、AIは、より洗練されたデザインに変更してくれました。

  • 余白を調整
  • フォントサイズを調整
  • 色のコントラストを調整

まるで、プロのデザイナーが手を加えたような、そんな仕上がりでした。

プログラミングはできなくても、ディレクターにはなれる

「デザインの知識がないから、指示できない」

そう思うかもしれません。でも、実は、知識がなくても大丈夫です。

  • 「かっこよくして」
  • 「プロっぽくして」
  • 「もっと見やすくして」
  • 「高級感を出して」

こういう、感覚的な言葉でも、AIは理解してくれます。

プログラミングはできなくても、デザインの「ディレクター」にはなれるのです。

段階的に調整する

でも、最初から完璧を求めすぎると、うまくいきません。

私は、こうやって段階的に調整していきました。

  1. まずは動かす → 「動けばいい」
  2. 見た目を整える → 「かっこよくして」
  3. 細部を調整する → 「もっと見やすくして」

最初は「動けばいい」と思っていました。でも、動いたら「もっとかっこよくしたい」と思いました。そして、かっこよくなったら「もっと見やすくしたい」と思いました。

こうして、段階的に改善していくことで、最終的には、自分が満足できるデザインになりました。

所有感が生まれる瞬間

デザインが完成した時、私は気づきました。

「これ、私のアプリだ」

URLをカスタマイズして、サービス名に変更した時、その感覚はさらに強くなりました。

「これが、私が作ったサービスだ」

プログラミングの知識がなくても、AIと対話することで、自分だけのアプリを作ることができる。

そして、そのアプリを、世界中の人に使ってもらえる。

この「所有感」こそが、アプリ開発の、最も大きな喜びなのかもしれません。

次のステップへ

デザインが完成し、Webに公開した。

これで、アプリ開発は終わり...ではありません。

次は、このWebアプリを、パソコンで動くアプリにする作業が待っています。

ブラウザを開かなくても、デスクトップのアイコンをダブルクリックするだけで、アプリが立ち上がる。
