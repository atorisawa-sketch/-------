第4章：資産化編 〜プログラムをパッケージにする〜

「node_modules」という巨大な壁

アプリが完成し、Webに公開した私は、友達に自慢したくなりました。

「これ、私が作ったアプリだよ。コードも見せてあげる」

そう言って、プロジェクトフォルダを、ZIPファイルに圧縮しようとしました。

「え、なにこれ...」

圧縮を始めた瞬間、私は驚きました。

「サイズが大きすぎます」

画面に表示されたエラーメッセージを見て、私は一瞬、理解できませんでした。

「なんで、こんなに大きいの？」

フォルダのサイズを確認すると、数GBもありました。まるで、映画のファイルを圧縮しようとしているような、そんな大きさです。

「コードって、こんなに大きいものなの？」

私は、AIに聞いてみました。

「なぜ、プロジェクトフォルダがこんなに大きいのですか？」

AIは、優しく教えてくれました。

「node_modules というフォルダが、巨大になっています。これは、アプリを動かすために必要な部品が入っているフォルダです。でも、これは送る必要はありません」

「送ってはいけないフォルダ」

AIは、続けて説明してくれました。

「GitHubにコードを送る時や、友達にコードを送る時は、node_modules フォルダは送らないでください。相手が npm install というコマンドを実行すれば、自動的にダウンロードされます」

なるほど。つまり、node_modules は、「送ってはいけないフォルダ」だったのです。

.gitignore という「見えないリスト」

AIは、.gitignore というファイルを作ってくれました。

「このファイルに、送ってはいけないフォルダやファイルの名前を書いておきます。そうすれば、GitHubにアップロードする時も、自動的に除外されます」

.gitignore の中身は、こんな感じでした。

node_modules/ .next/ .env.local

  • node_modules/ → 部品のフォルダ
  • .next/ → ビルド結果のフォルダ
  • .env.local → APIキーが入っているファイル（秘密なので送ってはいけない）

「これで、安全にコードを共有できます」

AIが作ってくれた .gitignore ファイルのおかげで、私は安心してコードを友達に送ることができるようになりました。

README.md：アプリの取扱説明書

未来の自分と友達のために

コードを整理したら、次は「説明書」です。

「これ、どうやって使うんだっけ？」

数週間後、数ヶ月後、自分で作ったアプリの使い方を忘れてしまうかもしれません。そして、友達がコードを見ても、「何これ？」と思われるかもしれません。

そんな時、README.mdという「取扱説明書」があれば、安心です。

AIに「かっこいい説明書」を書いてもらう

私は、AIにこう伝えました。

README.mdという、アプリの説明書を作ってください。 かっこよく、プロっぽく書いてください。

AIは、すぐに README.md というファイルを作ってくれました。

中身を見て、私は感動しました。

```

✍️ プロの添削くん

「AIを、あなたの専属編集者に。」 プロの編集者レベルの校閲・推敲を、あなたのブラウザから直接体験できるツールです。 ```

「これ、本当に私が作ったアプリの説明書？」

まるで、プロの開発者が書いたような、洗練された説明書でした。

説明書に含まれる内容

AIが作ってくれた説明書には、以下の内容が含まれていました。

  1. アプリの説明 → 何ができるのか
  2. 使い方 → どうやって使うのか（3ステップ）
  3. なぜ安心なのか → セキュリティの説明
  4. 開発者より → メッセージ

「これで、未来の自分も、友達も、すぐに理解できる」

私は、その説明書を見て、安心感に包まれました。もう、使い方を忘れる心配はありません。

説明書をカスタマイズする

でも、AIが作ってくれた説明書は、まだ「テンプレート」のような感じでした。

「もっと、私らしさを出したい」

私は、AIにこう伝えました。

README.mdを、もっと親しみやすい感じにしてください。 「初心者でも使えます」というメッセージを追加してください。

AIは、説明書を修正して、より親しみやすいトーンに変更してくれました。

「これで、誰でも使えるアプリだということが伝わる」

説明書をカスタマイズしたことで、アプリが、より「自分らしい」ものになりました。

資産としてのコード

コードは「資産」になる

コードを整理し、説明書を作った後、私は気づきました。

「これ、私の資産だ」

コードは、ただの文字列ではありません。時間と労力をかけて作った、価値のある資産です。

  • GitHubに保存すれば、どこからでもアクセスできる
  • 説明書があれば、いつでも使い方を思い出せる
  • 友達に送れば、一緒に使える

「これで、ずっと使えるアプリになった」

私は、GitHubにコードをアップロードし、説明書も一緒に保存しました。

次のステップへ

コードを整理し、説明書を作った。

これで、アプリは「資産」として、しっかりと保存されました。

でも、まだ終わりではありません。

次は、このWebアプリを、パソコンで動くアプリにする作業が待っています。

ブラウザを開かなくても、デスクトップのアイコンをダブルクリックするだけで、アプリが立ち上がる。

そんな夢のような話が、次の章で現実になります。
