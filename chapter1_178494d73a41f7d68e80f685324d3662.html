第1章：準備編 〜魔法の杖を手に入れる〜

魔法の杖、その名は「Cursor」

「アプリを作りたい」

その気持ちだけが、私を動かしていました。でも、どこから始めればいいのか。プログラミングの本を買う？ オンライン講座を受ける？ それとも、まずは「Hello World」から？

そんな迷いの中、ある日、インターネットで「Cursor」という名前を見つけました。

「AIがコードを書いてくれるエディタ」

その一行に、私の心は一気に引き寄せられました。ダウンロードボタンをクリックする指先が、なぜか震えていました。まるで、魔法の杖を手に入れる瞬間のように。

インストールは、思っていたより簡単でした。公式サイトからダウンロードして、インストーラーを実行するだけ。数分後、デスクトップに「Cursor」のアイコンが現れました。

そのアイコンをダブルクリックした瞬間、何かが変わる予感がしました。

Cursorの画面を理解する：最初の「迷子」体験

Cursorを開いた私は、画面を見て、一瞬、戸惑いました。

「どこに何があるの？」

真っ白な画面のどこかに、AIと話す場所があるはず。でも、どこをクリックすればいいのかわからない。まるで、新しい街に来て、地図もなく迷子になったような気持ちでした。

チャット画面を開く：AIと話す場所

実は、Cursorには2つの重要な画面があります。最初に知っておくと、迷子にならずに済みます。

方法1：キーボードショートカット（一番簡単）

  1. Ctrl + L キーを同時に押す（Macの場合は Cmd + L）
  2. 画面の右側に、チャット画面が現れます

これが、AIと会話する場所です。ここに「Next.jsでAI添削アプリを作って」と入力すれば、AIが答えてくれます。

方法2：サイドバーから開く

  1. 画面の左側にあるサイドバーを見てください
  2. チャットアイコン（吹き出しのマーク）をクリック
  3. 右側にチャット画面が表示されます

ターミナル画面を開く：コマンドを実行する場所

AIが「このコマンドを実行してください」と言ったとき、そのコマンドを打ち込む場所が「ターミナル」です。黒い画面で、文字が並んでいます。

方法1：キーボードショートカット（一番簡単）

  1. Ctrl + @ キーを同時に押す（@はShift + 2で入力できる記号）
    • または Ctrl + Shift + @
  2. 画面の下部に、黒いターミナル画面が現れます

方法2：メニューから開く

  1. 画面上部のメニューバーから「ターミナル」を選択
  2. 「新しいターミナル」をクリック
  3. 画面下部にターミナルが表示されます

私の最初の失敗談

実は、私は最初、チャット画面もターミナルも見つけられませんでした。

「AIと話すって言うけど、どこで話すの？」

画面をクリックしまくって、右クリックしてみて、メニューを探してみて...。結局、30分くらい迷子になっていました。

でも、Ctrl + L というキーボードショートカットを知った瞬間、すべてが変わりました。

「あ、ここにあったのか！」

この2つの画面さえ知っていれば、Cursorを使い始めることができます。もし今、あなたが迷子になっているなら、まずはこの2つのショートカットを覚えてください。

  • チャット画面: Ctrl + L（Macは Cmd + L）
  • ターミナル: Ctrl + @

これだけで、AIと会話して、コマンドを実行できるようになります。

最初の一歩：APIキーという「鍵」

Cursorを開いた私は、すぐに次の壁にぶつかりました。

「APIキーが必要です」

画面に表示された文字を見て、私は一瞬、凍りつきました。APIキー？ それは何？ どこで手に入れるの？

でも、ここで諦めるわけにはいきません。Google検索で「Gemini API キー 取得方法」と検索しました。すると、Google AI Studioというページが表示されました。

優しい手順：APIキーを手に入れる

  1. Google AI Studioにアクセス

    • ブラウザで https://aistudio.google.com/ を開きます
    • Googleアカウントでログインします（GmailのアカウントがあればOK）

  2. APIキーを作成

    • 画面右上の「Get API key」ボタンをクリック
    • 「Create API key」を選択
    • プロジェクトを選択（新規作成でもOK）
    • すると、AIza... で始まる長い文字列が表示されます

  3. 大切に保管

    • この文字列が、あなたの「魔法の鍵」です
    • コピーして、安全な場所に保存しておきましょう
    • ※注意：このキーは他人に見せてはいけません

最初は「こんな長い文字列、本当にこれで動くの？」と不安でした。でも、後でわかったことですが、この鍵こそが、AIと会話するための唯一の通行証だったのです。

魔法の瞬間：最初の指示

APIキーを手に入れた私は、Cursorの画面を前に、深呼吸をしました。

「さあ、何を書けばいいんだろう」

真っ白な画面を見つめながら、私はキーボードに指を置きました。そして、迷わず、こう打ち込みました。

Next.jsでAI添削アプリを作って

Enterキーを押した瞬間、画面に何かが起こりました。

コードが、溢れ出したのです。

npx create-next-app@latest . --yes

import { NextRequest, NextResponse } from 'next/server';

'use client'; import { useState } from 'react';

画面に現れた文字の羅列を見て、私は一瞬、圧倒されました。

「これは...何？ 全部読まないといけないの？」

でも、AIが優しく教えてくれました。

「まずは、このコマンドを実行してみてください」

画面に表示された npm run dev という文字を見て、私は Ctrl + @ を押してターミナルを開きました。そして、そのコマンドをコピー＆ペーストして、Enterキーを押しました。

数秒後、ブラウザが自動的に開き、「Welcome to Next.js」という画面が表示されました。

「動いた...動いた！」

私は、思わず声に出して叫んでいました。まだ何も作っていないのに、この感動は何だったのでしょう。まるで、魔法の杖が本当に動いた瞬間でした。

【コラム】コードの山に圧倒されても大丈夫

最初にコードを見たとき、多くの人がこう思うはずです。

「こんなにたくさんの文字...全部理解しないといけないの？」

答えは、「いいえ」です。

プログラマーではない私たちにとって、コードは「読むもの」ではなく、「AIと会話するための材料」です。

例えば、料理のレシピを見るとき、あなたは化学式を理解する必要はありません。「この材料を、この順番で混ぜる」という手順だけがわかれば、美味しい料理が作れます。

コードも同じです。

  • 「このファイルを開いて」
  • 「この部分を変更して」
  • 「このコマンドを実行して」

AIに指示を出すだけで、コードは動き出します。あなたが理解する必要があるのは、「何を作りたいか」という目的だけです。

私も最初は、画面に表示されたコードを見て、頭がクラクラしました。でも、AIに「これは何をしているの？」と聞けば、優しく説明してくれます。「この部分を変更したい」と言えば、すぐに修正してくれます。

「読む必要はない、話せばいい」

このマインドセットが、プログラミング初心者の私たちを、魔法使いに変えるのです。

次のステップへ

Cursorを開き、APIキーを手に入れ、最初のアプリが動いた。

この3つのステップを踏んだとき、私は気づきました。

「あれ、思っていたより簡単かも？」

もちろん、まだ何も作っていません。でも、魔法の杖は手に入れました。あとは、どう使うかだけです。

次の章では、実際に「プロの添削くん」を作り始めます。AIとの対話の中で、どのようにアプリが形になっていくのか。その過程を、一緒に見ていきましょう。
